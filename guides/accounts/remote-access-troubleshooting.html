This guide helps staff troubleshoot common remote access issues when working from home or connecting to PVNCCDSB network resources. Follow the solutions step by step for each problem you encounter.

Before You Start Troubleshooting

Before diving into specific issues, verify these basics:

  • ‚úÖ You have stable home internet connection (test at fast.com)
  • ‚úÖ Your PVNCCDSB account credentials are active and not expired
  • ‚úÖ You're using an approved device (board-issued or personal device with proper configuration)
  • ‚úÖ Any VPN software is up to date
  • ‚úÖ You know your username format: firstname.lastname@pvnccdsb.on.ca

VPN Connection Issues

VPN Won't Connect or Times Out

Solutions to Try:

  1. Check your internet connection:
    • Open a web browser and visit any website
    • If websites don't load, your internet is down - contact your ISP
    • Run a speed test at fast.com - you need at least 5 Mbps
  2. Restart the VPN client:
    • Close the VPN application completely
    • Wait 30 seconds
    • Reopen and try connecting again
  3. Check VPN server address:
    • Verify you're using the correct VPN server address provided by IT
    • Double-check for typos in the server name
  4. Try a different network:
    • If using Wi-Fi, try connecting via ethernet cable
    • Try your mobile hotspot to rule out ISP blocking VPN
  5. Disable other VPN or proxy:
    • Turn off any personal VPN services
    • Disable browser proxy settings
    • Close any VPN browser extensions

VPN Connects but Resources Not Accessible

Solutions to Try:

  1. Verify VPN is fully connected:
    • Check VPN status indicator shows "Connected"
    • Look for any warning messages in VPN client
  2. Disconnect and reconnect:
    • Fully disconnect from VPN
    • Wait 10 seconds
    • Reconnect and test access again
  3. Check split tunneling:
    • Ensure VPN is configured to route all traffic through PVNCCDSB network
    • Contact IT if you're unsure about split tunnel settings
  4. Test specific resources:
    • Try accessing different services (email, shared drives, internal websites)
    • Note which resources work and which don't when contacting IT

VPN Keeps Disconnecting

Solutions to Try:

  1. Check internet stability:
    • Unstable internet often causes VPN drops
    • Move closer to Wi-Fi router or use ethernet cable
    • Restart your modem and router
  2. Adjust power settings:
    • Prevent computer from going to sleep
    • Set network adapter to never turn off for power saving
  3. Update VPN client:
    • Check for VPN software updates
    • Install latest version if available
  4. Configure VPN timeout settings:
    • Check if VPN has idle timeout settings
    • Increase timeout duration in VPN settings if option exists

Authentication and Login Problems

"Invalid Username or Password" Error

Solutions to Try:

  1. Verify username format:
    • Use: firstname.lastname@pvnccdsb.on.ca
    • Do NOT use just "firstname.lastname" without domain
    • Check for typos or extra spaces
  2. Check Caps Lock:
    • Passwords are case-sensitive
    • Ensure Caps Lock is OFF
  3. Try password reset:
    • If password recently changed, wait 15 minutes for sync
    • Reset password at: Microsoft password reset page
    • Or contact IT Help Desk for password reset
  4. Check account status:
    • Your account may be locked after multiple failed login attempts
    • Contact IT Help Desk to unlock account

Multi-Factor Authentication (MFA) Not Working

Solutions to Try:

  1. Check Microsoft Authenticator app:
    • Ensure app is installed and updated
    • Open app to verify PVNCCDSB account is listed
    • Check phone has internet or cellular connection
  2. Use alternative authentication method:
    • Try "Text me a code" option
    • Use "Call me" if text doesn't arrive
    • Enter backup code if you have one saved
  3. Time sync issues:
    • Ensure phone date/time is set to automatic
    • Codes are time-sensitive and won't work if time is wrong
  4. Re-register MFA:
    • Contact IT Help Desk to reset MFA registration
    • Follow setup wizard to reconfigure authentication methods

Network and Connectivity Issues

Can't Access Network Drives or Shared Folders

Solutions to Try:

  1. Verify VPN connection:
    • Network drives require active VPN connection
    • Confirm VPN is connected before accessing drives
  2. Check drive path:
    • Use UNC path format: \\server\share
    • Don't use mapped drive letters (H:, S:, etc.) - they may not work remotely
  3. Reconnect network drives:
    • Open File Explorer
    • Right-click on disconnected drive ‚Üí Disconnect
    • Map drive again using UNC path with credentials
  4. Check permissions:
    • Verify you have permission to access the specific folder
    • Contact your supervisor or IT if you need access granted

"DNS Server Not Responding" or Website Won't Load

Solutions to Try:

  1. Flush DNS cache:
    • Open Command Prompt as Administrator
    • Type: ipconfig /flushdns
    • Press Enter and try accessing site again
  2. Restart network adapter:
    • Turn Wi-Fi off and back on
    • Or disable/enable network adapter in Network Settings
  3. Check DNS settings:
    • Ensure DNS is set to automatic (obtain automatically)
    • Don't manually configure DNS servers unless instructed by IT
  4. Test external sites:
    • Try accessing google.com or another external site
    • If external sites work but internal don't, it's VPN/DNS routing issue

Remote Desktop Connection Issues

Remote Desktop Connection Failed or Timed Out

Solutions to Try:

  1. Confirm VPN is connected:
    • Remote desktop REQUIRES active VPN connection first
    • Connect to VPN, then launch Remote Desktop
  2. Verify computer name or IP:
    • Double-check you're using correct computer name or IP address
    • Computer names are case-insensitive but must be spelled correctly
  3. Check if office computer is on:
    • Remote computer must be powered on and connected to network
    • Ask colleague to check if computer is on, or contact IT
  4. Test network connectivity:
    • Open Command Prompt
    • Type: ping computername
    • If ping fails, computer is unreachable via network
  5. Check Remote Desktop is enabled:
    • Remote Desktop must be enabled on target computer
    • Contact IT to verify RDP is enabled on your office PC

Remote Desktop Screen is Blank or Frozen

Solutions to Try:

  1. Press Ctrl+Alt+End:
    • This is equivalent to Ctrl+Alt+Delete on remote computer
    • Opens Task Manager to close frozen applications
  2. Disconnect and reconnect:
    • Close Remote Desktop window
    • Wait 30 seconds
    • Reconnect to session
  3. Reduce display quality:
    • In RDP settings, reduce color depth to 16-bit
    • Lower screen resolution
    • Disable desktop background and visual effects
  4. Check bandwidth:
    • Slow internet can cause freezing
    • Close bandwidth-heavy applications (streaming, downloads)

Performance and Speed Issues

Slow VPN or Remote Desktop Performance

Solutions to Try:

  1. Test internet speed:
    • Run speed test at fast.com
    • Minimum recommended: 10 Mbps download, 5 Mbps upload
    • Contact your ISP if speeds are consistently slow
  2. Use wired connection:
    • Connect computer directly to router via ethernet cable
    • Ethernet is faster and more stable than Wi-Fi
  3. Close unnecessary applications:
    • Close programs you're not using
    • Pause cloud sync services (OneDrive, Dropbox, etc.)
    • Stop video streaming or large downloads
  4. Optimize Remote Desktop settings:
    • Reduce screen resolution
    • Disable visual effects and themes
    • Turn off printer redirection if not needed
    • Disable clipboard sharing temporarily
  5. Connect during off-peak hours:
    • Network can be slower during high-usage times (9am-3pm)
    • Try working early morning or evening if flexibility allows

General Troubleshooting Tips

  • Restart first: Many issues resolve with a simple restart of computer, router, or VPN
  • Document the issue: Note error messages, screenshots, and steps you took before problem occurred
  • Check for outages: Ask colleagues if they're experiencing similar issues - might be system-wide
  • Keep software updated: Install Windows updates and VPN client updates regularly
  • Test at different times: If issue only occurs at specific times, note this when contacting IT

Still Having Issues? Contact IT Support

If you've tried the solutions above and still can't connect, contact the IT Help Desk with:

  • Detailed description of the problem
  • Exact error messages (screenshots helpful)
  • Steps you've already tried
  • When the problem started
  • Your internet provider and approximate speed
