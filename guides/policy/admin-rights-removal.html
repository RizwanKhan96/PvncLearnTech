As part of PVNCCDSB's commitment to cybersecurity and network protection, administrator rights are being removed from student and staff computers. This document explains why and what it means for you.

What Are Administrator Rights?

Administrator rights (or "admin rights") give users the ability to:

  • Install and uninstall software programs
  • Make system-level changes to the computer
  • Modify security settings and policies
  • Access and modify system files
  • Change settings that affect all users on the device

When administrator rights are removed, these capabilities are restricted to ensure device security and compliance with board policies.

Why This Policy Exists

1. Enhanced Security

Removing admin rights significantly improves the security of our network and devices:

  • Prevents malware installation: Most malware requires admin rights to install. Without these rights, malicious software cannot easily infiltrate devices.
  • Blocks unauthorized software: Prevents installation of unapproved applications that may contain security vulnerabilities.
  • Protects system files: Critical operating system files remain protected from accidental or intentional modification.
  • Reduces ransomware risk: Limits the spread and impact of ransomware attacks.

2. Legal and Policy Compliance

  • Software licensing: Ensures only properly licensed software is installed on board devices
  • Content filtering: Maintains required web filtering and monitoring for student safety
  • Privacy regulations: Protects student data in accordance with privacy legislation (MFIPPA)
  • Acceptable use policies: Enforces board technology acceptable use policies

3. Network Protection

  • Prevents network disruption: Stops installation of peer-to-peer software or network scanning tools
  • Bandwidth management: Prevents unauthorized applications from consuming network resources
  • Consistent configuration: Maintains standardized security settings across all devices

4. Device Stability

  • Reduces technical issues: Prevents installation of conflicting or unstable software
  • Easier troubleshooting: IT can diagnose issues more quickly with standardized configurations
  • Longer device lifespan: Devices remain functional longer without system corruption

Did You Know?

According to cybersecurity research, removing administrator rights can prevent up to 94% of critical Windows vulnerabilities from being exploited. This single policy change is one of the most effective security measures an organization can implement.

What If I Need Software Installed?

If you require software for educational or work purposes, follow this process:

Step 1: Check if It's Already Available

  • Look in the Start Menu for installed applications
  • Check the Software Center (if available on your device)
  • Explore web-based alternatives (many tools have online versions)
  • Ask colleagues if they use a similar tool that's already approved

Step 2: Submit a Software Request

If the software isn't available, submit a request:

  • Staff: Submit a request through the IT Help Desk portal
  • Students: Ask your teacher to submit a request on your behalf
  • Include in your request:
    • Software name and version
    • Educational/business purpose
    • Subject area or department need
    • Number of users who need it
    • Licensing information (if known)

Step 3: IT Review Process

The IT department will evaluate your request based on:

  • Educational value: Does it support curriculum or work requirements?
  • Security assessment: Is the software from a reputable source?
  • Licensing compliance: Can we legally install it on board devices?
  • Technical compatibility: Will it work with our systems?
  • Cost considerations: Is there a free or lower-cost alternative?

Step 4: Installation

If approved, IT will:

  • Purchase necessary licenses (if required)
  • Test the software for compatibility and security
  • Install it on requested devices or make it available through Software Center
  • Provide basic usage instructions or training resources

Request Turnaround Time

Typical processing time: 5-10 business days for straightforward requests

Complex requests: May take 2-3 weeks if licensing or extensive testing is required

Urgent needs: Contact the IT Help Desk directly to explain the urgency

Common Issues and Solutions

"I Can't Install a Browser Extension"

Solution: Most educational Chrome extensions can be requested through IT. Many extensions are pre-approved and can be installed by IT staff quickly. Submit a request with the extension name and URL.

"A Program Won't Update"

Solution: Software updates are managed centrally by IT to ensure compatibility. If you see an update notification, the update will be deployed automatically during maintenance windows. For urgent updates, contact IT.

"I Need to Install Drivers for a Device"

Solution: Most standard devices (printers, mice, keyboards) will work without manual driver installation. For specialized equipment (3D printers, science probes, etc.), submit an IT request including the device make and model.

"The Software I Need Requires Admin Rights"

Solution: Some legitimate educational software does require elevated permissions. IT can install these applications and configure them to run without admin rights, or create a temporary elevated account for specific uses.

"I'm Getting 'Access Denied' Errors"

Solution: This is expected behavior when trying to perform administrator-level tasks. If you believe you should have access to the function, submit an IT ticket describing what you're trying to do and why it's needed for your work.

Best Practices for Working Without Admin Rights

  • Use web-based tools: Many applications have browser versions that don't require installation (Google Workspace, Microsoft Office Online, Canva, etc.)
  • Explore portable apps: Some programs can run without installation (must still be approved by IT)
  • Plan ahead: Submit software requests well before you need the tool for a project or lesson
  • Document needs: Keep a list of software you use regularly so IT can ensure it's available
  • Communicate: Let IT know about upcoming projects that may require new tools
